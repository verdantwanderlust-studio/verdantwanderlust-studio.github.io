# Ivy 跟 AI 助理對話速查表

> 給 Ivy 看的「我可以怎麼開口」範例集。完全用中文,口語化。
> 想做某件事但不知道怎麼講?來這找最接近的例子。

---

## 大原則:像對助理交辦事情那樣講話

❌ 不要這樣:「請執行 add-photo 指令」
✅ 這樣比較好:「我有 3 張新的作品照片要放上去」

❌ 不要這樣:「edit src/content/index.mdx hero section」
✅ 這樣比較好:「首頁那段歡迎詞我想換成更溫暖的版本」

AI 會自己找對的檔案、判斷該觸發哪個 skill。**你只負責想清楚要什麼**。

---

## 改文字 / 文案

```
「首頁的標語想換成『為每個花園找到最自然的呼吸』」
「about 那頁的自我介紹太正式了,改溫暖一點,口語一點」
「服務頁面那段『庭園規劃設計』下面的描述,我想加一句強調我們有 10 年經驗」
「把作品集那篇『日式枯山水』的標題改成『靜謐的禪意花園』」
「contact 頁的文字幫我看一下,有沒有更親切的寫法?」
```

**小撇步**:
- 不確定怎麼改 → 跟 AI 講你想要的「感覺」,讓它先給 2-3 個版本讓你挑
- 想保留某幾個字 → 直接講「『XX』這幾個字一定要保留」

---

## 加 / 換照片

```
「我有 5 張新的庭園案例要放作品集」
「首頁主視覺想換,我準備了 3 張讓你選」
「about 頁我自己的大頭照要換,新的拍好了」
「『日式禪園』那個案例只有 1 張圖太單調,我加了 4 張細節照」
「服務頁的『庭園維護』那個區塊配圖我覺得不對味,有更貼切的嗎?」
```

**上傳方式**:
- AI 會引導你用 GitHub 介面拖拉上傳到 `raw-images/` 對的子資料夾
- **檔名請用英文 + dash**(`japanese-garden-2025.jpg`),不要中文或空格
- 原圖原尺寸丟即可,優化的事 AI 處理

---

## 微調顏色 / 字體 / 視覺

```
「主色想換成更暖一點的綠,現在這個太冷」
「標題字感覺太細,可以粗一點嗎?」
「首頁段落之間太擠,撐開一點」
「按鈕的圓角太硬,軟一點」
「整體想要再素一點,現在感覺有點花」
「跟 [reference 網站 URL] 那種感覺靠近,但保留我們的綠色」
```

**邊界**:
- ✅ 可以調:**顏色 value、字級數字、間距數字、圓角數字**
- ❌ 不能調:layout 結構(把 hero 移到底下、改成兩欄、加 sidebar)→ 這要找 Jason

---

## 加新頁 / 加新案例

```
「想加一頁專門講『花園維護年費方案』」
「作品集想加一個新案例:『2026 春 — 大安區陽台改造』」
「FAQ 頁想要,可以幫我建嗎?幾個常見問題我會給你」
「想加一個『部落格』區放我的隨筆」
```

**AI 會問**:
- 這頁的名字 / URL slug
- 要放在導覽列哪裡
- 大致內容方向

---

## 上線 / 預覽

```
「上線」
「把這些改動推上去」
「先給我看 preview 不要正式上線」
「OK 沒問題,正式發布」
```

**preview vs production**:
- **小改**(改一句話、換一張內頁圖):AI 直接 push 到 main → 30 秒後正式上線
- **大改**(改首頁、改主視覺、加新頁):AI 會先建 preview branch → 給你 Cloudflare preview URL 看 → 你確認再 merge

---

## 看狀態 / 確認

```
「網站還好嗎?」
「最近一次上線是什麼時候?改了什麼?」
「圖片用了多少空間?還有多少?」
「網站打不開了!」(這個直接找 Jason,別跟 AI 卡)
```

---

## 求救 / 反悔

```
「剛剛那個改動我想 undo」
「我反悔了,改回原本的」
「這個版本不好,還有其他版本嗎?」
「比較看看『A 版』跟『B 版』」
```

**重要**:Cloudflare 會保留 1 個月的 deploy 歷史,理論上任何改動都能 rollback。緊張的時候找 Jason。

---

## 不確定怎麼開口?

直接問 AI:

```
「我想要改網站但不確定怎麼講,可以給我幾個例子嗎?」
「我想做 XXX,你覺得這算改文字、改設計、還是加新功能?」
「這個事情你能做嗎?如果不行,Jason 才能做嗎?」
```

---

## AI 會說「找 Jason」的情況

碰到下面這些 AI 會推回給 Jason:

- 想改 layout 結構(欄位數、位置、順序大改)
- 想加 form / 訂閱 / 報名功能
- 想加會員系統
- 網站壞了 / build 紅燈
- 想做品牌大改 / 重新設計

**這時候**:AI 會幫你把需求整理成一段文字,你 copy 給 Jason 即可,不用自己重講。

## 對話範例(完整一次互動長這樣)

> **Ivy**: 我有 4 張新的庭園案例要放作品集,主題是「小坪數陽台」
>
> **AI**: 好的,我幫你加 4 張到作品集的「小坪數陽台」案例。先確認一下:這是新增一個案例(就是作品集多一個項目),還是現有的某個案例補圖?
>
> **Ivy**: 新的,案例名字叫做「2026 春 — 大安區陽台」
>
> **AI**: 了解。我會:
> 1. 在 `src/content/portfolio/` 新增 `dazhi-balcony-2026.mdx`
> 2. 引導你把 4 張圖上傳到 `raw-images/portfolio/dazhi-balcony-2026/`
> 3. GitHub Action 30 秒內優化完
> 4. 我把 MDX 引用設好
> 5. 推上線,給你 preview URL
>
> 案例的標題、描述文字你要先給我,還是我先寫個草稿你改?
>
> **Ivy**: 先寫草稿給我吧,主題是「在 5 坪陽台種出 30 種植物的多層次小宇宙」
>
> **AI**: [生成草稿] ...
>
> ... (繼續直到完成)
